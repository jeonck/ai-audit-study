# AI 기술 이해도

AI 기술 이해도는 AI 시스템을 개발, 운용, 감시하는 데 필요한 기술적 역량과 개념을 의미합니다. 이는 AI 시스템이 신뢰성과 안전성을 확보하고 윤리적으로 운영되도록 하는 데 중요한 역할을 합니다.

## 1. AI 모델 개발 및 최적화

| 개념 | 설명 | HITL과의 관계 |
|------|------|---------------|
| **파인튜닝 (Fine-tuning)** | 사전 학습 모델을 특정 작업이나 도메인에 맞게 추가 학습시키는 과정. | HITL은 파인튜닝을 위한 고품질 레이블링된 데이터를 지속적으로 제공하여 모델 성능 향상에 기여합니다. |
| **프롬프팅 (Prompting)** | LLM에 원하는 결과나 행동을 유도하기 위해 입력하는 명령어 또는 지시. | HITL은 모델이 이해하기 어렵거나 모호한 프롬프트에 대해 피드백을 제공하거나, 프롬프트 결과를 사람이 직접 평가하는 데 사용될 수 있습니다. |
| **RAG (Retrieval-Augmented Generation)** | 외부 지식 기반에서 관련 정보를 검색하고, 이 정보를 기반으로 답변을 생성하는 기술. | HITL은 RAG가 검색한 정보가 답변을 생성하는 데 적절하고 정확한지를 검증하여 모델의 신뢰도를 높이는 역할을 합니다. |

## 2. HITL (Human-In-The-Loop) 구조

HITL은 인간의 판단력과 개입을 AI 시스템의 학습 및 의사결정 과정에 통합하는 설계 방법론입니다.

### 정의
AI 모델이 자신 없는 예측을 하거나, 중요하고 민감한 의사결정이 필요한 순간에 작업을 사람에게 이관하여 검토 및 수정하게 하는 시스템 구조입니다.

### 주요 역할
- **데이터 레이블링 및 검증**: AI 모델 학습에 필요한 고품질 데이터를 사람이 직접 레이블링하고 검토합니다.
- **모델 오류 수정**: 모델이 틀린 예측을 했을 때, 사람이 정답을 제공하여 모델이 재학습할 수 있는 **피드백 루프(Feedback Loop)**를 만듭니다.
- **예외 처리**: AI가 처리하기 어려운 **경계 사례(Edge Cases)**나 높은 위험을 수반하는 상황을 사람이 최종적으로 결정합니다.

### 장점
- **정확도 향상**: 사람의 전문 지식을 활용하여 모델의 정확도를 빠르게 높일 수 있습니다.
- **신뢰성 확보**: 특히 의료, 금융 등 고위험 분야에서 법적/윤리적 책임을 사람이 최종적으로 지게 하여 신뢰를 확보합니다.
- **편향 감소**: 사람의 개입을 통해 모델의 편향이나 불공정성을 감지하고 수정할 수 있습니다.

## 3. 데이터 처리 및 관리

| 개념 | 설명 |
|------|------|
| **전처리 과정 (Preprocessing)** | 원시 데이터를 모델이 학습할 수 있도록 정제, 변환, 표준화하는 단계. |
| **특성 공학 (Feature Engineering)** | 도메인 지식을 활용하여 모델 성능을 향상시키는 새로운 특성을 만들어내는 과정. |
| **임베딩 (Embedding)** | 복잡한 데이터를 저차원의 벡터 공간으로 변환하여 유사도를 측정 가능하게 함. |
| **벡터 DB (Vector Database)** | 임베딩된 데이터를 효율적으로 저장하고 벡터 유사도 검색을 수행하는 데이터베이스. |
| **데이터 파이프라인 (Data Pipeline)** | 데이터가 수집, 전처리, 저장, 분석되는 과정을 자동화하고 체계화한 흐름. |

## 4. AI 모델 운영 및 평가 & 통계 역량

- **MLOps (Machine Learning Operations)**: AI/ML 모델을 개발부터 배포, 운영, 모니터링하는 전 과정을 자동화하고 표준화하는 엔지니어링 방식.
- **성능 지표 (Performance Metrics)**: 모델의 성능을 정량적으로 측정하는 기준 (Accuracy, Precision, Recall, F1-Score, MSE 등).
- **분석 도구 (Analysis Tools)**: 데이터 탐색, 시각화, 통계적 분석을 수행하는 소프트웨어 및 라이브러리 (Pandas, Scikit-learn 등).
- **통계 역량**: 데이터 분석 및 모델 평가를 위한 통계적 방법론 (기술 통계, 가설 검증, 회귀 분석)을 이해하고 적용하는 능력.

## 5. 위험/윤리/보안 요소 반영 설계

| 위험 요소 | 설명 | HITL과의 연관성 |
|-----------|------|----------------|
| **모델 편향 (Model Bias)** | 학습 데이터의 편향으로 인한 불공정한 예측/결과. | HITL을 통해 사람이 편향된 출력을 검토하고 수정함으로써 편향성 완화에 기여. |
| **환각 (Hallucination)** | LLM이 사실이 아닌 정보를 그럴듯하게 생성하는 현상. | HITL이 생성된 답변의 사실 여부를 검증하여 환각을 즉시 잡아내고 피드백을 제공. |
| **프롬프트 인젝션 (Prompt Injection)** | 악의적인 명령으로 모델의 지침을 무시하거나 정보 유출을 유도하는 공격. | HITL 시스템이 의심스러운 프롬프트와 출력을 보안 관점에서 분석하고 차단할 수 있음. |
| **데이터 유출/개인정보보호** | 학습 및 서비스 과정에서 민감 정보가 외부에 노출되는 위험. | HITL은 데이터가 모델로 전달되거나 출력되기 전 사람이 민감 정보를 최종적으로 비식별화했는지 확인하는 관문 역할 수행. |
| **위험/윤리/보안요소 반영 설계** | AI 시스템 개발 초기부터 위험 예측 및 윤리 원칙 준수를 위한 안전 장치와 감사 체계를 통합하는 접근 방식. | HITL은 이 설계의 핵심 요소로, **책임감(Accountability)**과 **투명성(Transparency)**을 확보하는 수단입니다.
